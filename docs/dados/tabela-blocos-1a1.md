# Blocos 1:1

Cada bloco representa uma **etapa ou conjunto de informações** da planilha original.

## Padrão

- relacionamento OneToOne com `sig_registro`
- mesma chave primária (UUID)
- criação opcional na fase inicial

## Exemplos de blocos

- andamento
- financeiro
- proposta comercial
- projeto executivo
- instalação
- aprovação concessionária
- comissionamento

Esse modelo preserva a lógica original e permite evolução futura.
